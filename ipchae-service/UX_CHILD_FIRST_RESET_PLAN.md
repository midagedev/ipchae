# IPCHAE Child-First UX Reset Plan

기준일: 2026-02-28
대상: `/Users/hckim/repo/ipchae/ipchae-service`

## 1. 왜 리셋이 필요한가
현재 Studio는 기능 밀도가 높아서 초등학생 기준에서 초기 인지 부하가 크다.
1. 동시에 보이는 버튼/패널 수가 많음
2. 용어가 영어/전문어 중심(Validate, Slice, Sync 등)
3. "첫 3분 행동"이 명확하지 않음

## 2. 원칙 (초등 사용자 기준)
1. 한 화면에서 선택지는 3개 안팎으로 제한
2. 용어는 행동 중심 한국어로 단순화(`그리기`, `지우기`, `내보내기`)
3. 실수 복구는 즉시 보장(항상 `실행취소` 노출)
4. 고급 기능은 숨기고 필요 시 열기(기본: 초등 모드)
5. 매 단계 목표를 문장으로 제시(무엇을 하면 다음인지 명확히)

## 3. UX 구조 (2계층)
## 3.1 초등 모드 (기본)
1. 상단: `실행취소`, `지우기`, `내보내기`
2. 좌측: `Starter`, `Draw/Fill/Erase`, `Mirror`
3. 우측: `브러시 굵기`, `색상`, 저장 상태 메시지
4. 가이드: `1. 그리기 -> 2. 꾸미기 -> 3. 내보내기`

## 3.2 고급 모드
1. Import/Share/Save Part/Validate/Slice/View/Input 전체 노출
2. 협업/검증/동기화 디버그성 정보 유지

## 4. 이번 1차 반영 내용
1. Studio에 초등 모드 기본값 적용
2. 모드 토글 추가(`초등 모드`/`고급 모드`, localStorage 기억)
3. 초등 모드에서 고급 패널(View/Input/Slicer/Validate 상세) 숨김
4. 버튼 라벨 일부 한국어 단순화
5. E2E 시나리오를 새 UX 흐름으로 갱신
6. 주요 메뉴 아이콘화(홈 Quick Start/Service Tracks, Studio 상단 주요 액션)
7. 다국어 토글 도입(한국어/English/日本語, 로컬 저장)

## 5. 다음 반영(2차) 백로그
1. 홈 화면에 `초등 시작` 진입 카드 분리
2. Studio 온보딩 오버레이(첫 60초 튜토리얼)
3. 실수 방지 UX: 중요한 버튼(지우기/초기화) 2단계 확인
4. 보상 피드백 단순화(레벨/XP 대신 "오늘 만든 것" 중심)
5. 부모/교사용 보조 모드(고급 기능 고정 노출 + 저장/공유 단축)
6. 아이콘+텍스트 조합의 인지성 테스트(초등 고학년/중학생 각각)
7. 다국어 용어 표준집 작성(기능명, 상태문구, 경고문구)

## 6. 측정 지표 (초등 UX 성공 기준)
1. 첫 드로잉 시작 시간(TTFD) 20초 이하
2. 첫 내보내기 도달 시간 10분 이하
3. 세션 중 도움말 이동 비율 감소
4. `Clear` 직후 `Undo` 성공률 95% 이상
